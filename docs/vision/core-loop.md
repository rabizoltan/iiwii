# Core Loop

## Loop Summary

`Town -> Mission -> Extract or Push -> Resolve Outcome -> Progression -> Next Mission`

## Phase Breakdown

1. Town prep
- Choose class, loadout, and active mission.
- Spend or save currency/materials.

2. Mission entry
- Team spawns into procedural or authored encounter path.
- Complete objectives while collecting loot/resources.

3. Risk decision points
- Extract now for guaranteed value.
- Push deeper for higher rewards and higher death chance.

4. Run resolution
- Extraction: bank mission rewards and progression gains.
- Death/failure: lose run-carried resources, keep account progression layers.

5. Persistent progression
- Apply town upgrades and weapon mastery unlocks.
- Reconfigure build and queue next run.

## Design Requirements

- Every run has at least two explicit risk decision moments.
- Extraction points are telegraphed and contestable.
- Progression gain per run is visible before launching next mission.
