# Death and Extraction

## Outcome Philosophy
Failure must be meaningful but not account-invalidating.

## Role Of This Doc
- Defines mission-end outcome rules only.
- Use the progression docs for ownership of hero XP, weapon mastery, talents, and town progression.
- Use the ADR layer for the authoritative permadeath and mission-continuity rules.

## Extraction Rules
- Extraction is an explicit action at defined points/events.
- Successful extraction secures run rewards and progression gains.
- Staying longer increases potential rewards and death risk.

## Death Rules
- In-run death is permanent for that mission attempt in default mode.
- Run-carried loot/resources may be lost per loss table.
- Persistent town progression and weapon mastery are retained.

## Co-op Considerations
- Downed/revive state can exist before final death (tuning dependent).
- Team wipe resolves mission as failure.
- End-of-run summary must clearly separate lost run loot vs retained persistent gains.
