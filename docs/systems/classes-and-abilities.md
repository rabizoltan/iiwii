# Classes and Abilities
Category: Gameplay System
Role: Reference Contract
Last updated: 2026-03-16
Last validated: pending

## Purpose
- Describe the target class and ability model.
- This is a future-facing design doc, not the main current implementation driver.

## Baseline Role Shapes
- Frontliner: space control and survivability.
- Striker: burst damage and target deletion.
- Support: sustain, utility, and crowd control.

## Ability Model
- Active abilities: cooldown-based combat actions.
- Passive traits: class identity modifiers.
- Team utility skill per class to reinforce co-op synergy.

## Current Scope Note
- The current milestone focuses on movement, combat feel, and enemy navigation.
- Use this doc for direction only unless the task explicitly touches class identity or ability structure.

## Constraints
- Avoid mandatory single-class mechanics in v1.
- Abilities must communicate clearly in multiplayer readability tests.
- Host validates any ability effect with gameplay impact.
